# 在线预约挂号系统数据库设计

## 1. 设计基准

本项目数据库文档以以下文件为基准：

- DDL 主基准：`backend/数据库初始化脚本/db.sql`
- 种子数据：`backend/数据库初始化脚本/data.sql`
- 运行时对齐校验：`backend/src/main/java/com/hospital/registration/model/entity/**` + `mapper/**`

命名规范采用 `t_*` 前缀，核心业务表为：

- `t_user`（用户账号）
- `t_patient`（就诊人）
- `t_doctor`（医生）
- `t_schedule`（排班/号源）
- `t_appointment`（预约记录）

辅助表：

- `t_department`（科室）
- `t_notice`（公告）

## 3. 核心表结构说明

### 3.1 用户表 `t_user`

用途：承载登录账号与角色权限（患者/医生/管理员）。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | bigint unsigned | PK, AUTO_INCREMENT | 主键 |
| `username` | varchar(256) | NOT NULL | 用户名 |
| `password` | varchar(256) | NOT NULL | BCrypt 密文 |
| `phone` | varchar(20) | NOT NULL, UK(`uk_phone`) | 手机号 |
| `email` | varchar(64) | NOT NULL, UK(`uk_email_deletion_time`) | 邮箱 |
| `user_type` | varchar(20) | NOT NULL, default `PATIENT` | 角色值；代码实际使用 `PATIENT/DOCTOR/ADMIN/BAN` |
| `deletion_time` | bigint | NOT NULL, default `0` | 注销时间戳（0=未注销） |
| `del_flag` | tinyint(1) | NOT NULL, default `0` | 逻辑删除标识 |
| `create_time` | datetime | NOT NULL | 创建时间 |
| `update_time` | datetime | NOT NULL | 更新时间 |

补充：`uk_username_deletion_time (username, deletion_time)` 支持“未注销唯一，注销后可复用用户名”。

### 3.2 就诊人表 `t_patient`

用途：用户维护多就诊人，用于预约挂号。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | bigint unsigned | PK, AUTO_INCREMENT | 主键 |
| `user_id` | bigint unsigned | NOT NULL, FK(`fk_patient_user`) | 所属用户 |
| `real_name` | varchar(50) | NOT NULL | 姓名 |
| `id_card` | varchar(18) | NOT NULL, UK(`uk_id_card`) | 身份证号 |
| `phone` | varchar(20) | NULL | 联系方式 |
| `gender` | tinyint(1) | NOT NULL | 0=女，1=男 |
| `birth_date` | date | NULL | 出生日期 |
| `is_default` | boolean | NOT NULL, default `false` | 是否默认就诊人 |
| `create_time` | datetime | NOT NULL | 创建时间 |
| `update_time` | datetime | NOT NULL | 更新时间 |

### 3.3 医生表 `t_doctor`

用途：医生档案、科室归属、医生账号绑定。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | bigint unsigned | PK, AUTO_INCREMENT | 主键 |
| `department_id` | bigint unsigned | NOT NULL, FK(`fk_doctor_department`) | 所属科室 |
| `user_id` | bigint unsigned | NULL, FK(`fk_doctor_user`), UK(`uk_doctor_user_id`) | 绑定登录账号（可为空） |
| `name` | varchar(50) | NOT NULL | 姓名 |
| `title` | varchar(20) | NOT NULL | 职称（CHIEF/ASSOCIATE_CHIEF/ATTENDING/RESIDENT...） |
| `specialty` | varchar(200) | NULL | 专长 |
| `introduction` | text | NULL | 简介 |
| `avatar_url` | varchar(255) | NULL | 头像 |
| `registration_fee` | decimal(19,2) | NOT NULL, default `0.00` | 挂号费 |
| `del_flag` | tinyint(1) | NOT NULL, default `0` | 逻辑删除标识 |
| `create_time` | datetime | NOT NULL | 创建时间 |
| `update_time` | datetime | NOT NULL | 更新时间 |

### 3.4 号源表 `t_schedule`

用途：医生在某日某时段的可预约库存。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | bigint unsigned | PK, AUTO_INCREMENT | 主键 |
| `doctor_id` | bigint unsigned | NOT NULL, FK(`fk_schedule_doctor`) | 医生 ID |
| `schedule_date` | date | NOT NULL | 排班日期 |
| `time_slot` | varchar(10) | NOT NULL | `MORNING/AFTERNOON` |
| `total_amount` | int | NOT NULL | 总号源 |
| `remaining_amount` | int | NOT NULL | 剩余号源 |
| `status` | varchar(20) | NOT NULL, default `AVAILABLE` | `AVAILABLE/FULL/SUSPENDED` |
| `version` | bigint | NOT NULL, default `0` | 乐观锁版本号（预留） |
| `create_time` | datetime | NOT NULL | 创建时间 |
| `update_time` | datetime | NOT NULL | 更新时间 |

### 3.5 预约记录表 `t_appointment`

用途：预约主业务单，连接用户、就诊人、医生、排班与科室。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | bigint unsigned | PK, AUTO_INCREMENT | 主键 |
| `appointment_no` | varchar(20) | NOT NULL, UK(`uk_appointment_no`) | 预约号 |
| `user_id` | bigint unsigned | NOT NULL, FK(`fk_appointment_user`) | 下单用户 |
| `patient_id` | bigint unsigned | NOT NULL, FK(`fk_appointment_patient`) | 就诊人 |
| `doctor_id` | bigint unsigned | NOT NULL, FK(`fk_appointment_doctor`) | 医生 |
| `department_id` | bigint unsigned | NULL, FK(`fk_appointment_department`) | 冗余科室（用于约束/查询） |
| `schedule_id` | bigint unsigned | NOT NULL, FK(`fk_appointment_schedule`) | 排班 ID |
| `appointment_date` | date | NOT NULL | 就诊日期 |
| `time_slot` | varchar(10) | NOT NULL | 就诊时段 |
| `status` | varchar(20) | NOT NULL, default `PENDING` | `PENDING/COMPLETED/CANCELLED/NO_SHOW`（当前代码主要用前三种） |
| `queue_number` | int | NULL | 排队号 |
| `remarks` | varchar(500) | NULL | 备注 |
| `refuse_time` | datetime | NULL | 取消时间 |
| `refuse_reason` | varchar(200) | NULL | 取消原因 |
| `notification_sent` | boolean | NOT NULL, default `false` | 通知发送标记 |
| `create_time` | datetime | NOT NULL | 创建时间 |
| `update_time` | datetime | NOT NULL | 更新时间 |

补充：`db.sql` 通过 `ALTER TABLE` 给 `t_appointment` 增加了 `department_id`、`idx_dept_date`、`uk_patient_dept_date`。

## 5. 关键索引设计考虑

### 5.1 用户唯一性与软删除

- `uk_username_deletion_time (username, deletion_time)`
- `uk_email_deletion_time (email, deletion_time)`
- `uk_phone (phone)`

作用：兼顾账号唯一性与注销后复用场景。

### 5.2 排班防重复与高频查询

- `uk_doctor_date_slot (doctor_id, schedule_date, time_slot)`：防止同一医生同日同时间段重复排班。
- `idx_doctor_date (doctor_id, schedule_date)`：支撑“医生近7天/未来排班”查询。

### 5.3 预约主链路

- `uk_appointment_no (appointment_no)`：预约号唯一定位。
- `idx_user_id (user_id)`：我的预约列表。
- `idx_patient_id (patient_id)`：就诊人维度查询。
- `idx_doctorid_appointmentdate (doctor_id, appointment_date)`：医生工作台查询。
- `idx_dept_date (department_id, appointment_date)`：按科室日期统计与检索。
- `uk_patient_dept_date (patient_id, department_id, appointment_date)`：同就诊人同科室同日防重复。

### 5.4 约束策略说明

`uk_patient_dept_date` 不区分预约状态：取消记录仍占用唯一性。若未来希望“取消后同日可重新预约”，需要调整唯一键策略或迁移到应用层规则。

## 6. 数据一致性设计（与代码联动）

1. 事务边界  
创建预约与取消预约均在事务中处理（`AppointmentServiceImpl`），保证“预约记录与号源库存”同提交或同回滚。

2. 库存原子扣减  
`UPDATE t_schedule SET remaining_amount = remaining_amount - 1 WHERE id = ? AND remaining_amount > 0`，避免并发下负库存。

3. 状态联动  
扣减至 0 后排班置 `FULL`；取消回补后可从 `FULL` 恢复 `AVAILABLE`。

4. 规则双保险  
应用层先校验“同就诊人同科室同日不可重复”；数据库层再由 `uk_patient_dept_date` 兜底。
