# 运行指南

> 适用项目：`hospital-registration-system`  
> 更新时间：2026-04-27  
> 本文基于当前实际代码配置（`backend/src/main/resources/application.yml`、`frontend/vite.config.ts`）

## 1. 环境要求

| 软件 | 建议版本 | 说明 |
| --- | --- | --- |
| JDK | 17+ | 后端运行环境 |
| Maven | 3.8+ | 后端构建/启动 |
| Node.js | 18+ | 前端运行环境 |
| MySQL | 8.0+ | 数据库 |
| Yarn | 1.22+ / 4+ | 前端包管理与启动（本指南使用 Yarn） |

## 2. 默认端口与地址

| 服务 | 地址 | 配置来源 |
| --- | --- | --- |
| 前端开发服务 | `http://localhost:5173` | `frontend/vite.config.ts` |
| 后端服务 | `http://localhost:8080` | `backend/src/main/resources/application.yml` |
| Swagger UI | `http://localhost:8080/swagger-ui/index.html` | Springdoc |
| API 基础前缀 | `http://localhost:8080/api` | Controller 路由 |

## 3. 第一步：初始化数据库

### 3.1 建库与建表

执行：

- `backend/数据库初始化脚本/db.sql`

该脚本会创建数据库 `or_sys` 并建表。

### 3.2 导入初始数据（重要）

`backend/数据库初始化脚本/data.sql` 里默认写的是：

```sql
USE hospital_registration;
```

而当前项目后端连接库名是 `or_sys`。  
导入前请二选一：

1. 把 `USE hospital_registration;` 改成 `USE or_sys;`
2. 或者在 SQL 客户端先手动切到 `or_sys` 后再执行数据插入部分

## 4. 第二步：启动后端

在项目根目录打开终端：

```bash
cd backend
mvn spring-boot:run
```

或打包后运行：

```bash
cd backend
mvn clean package -DskipTests
java -jar target/registration-system-0.0.1-SNAPSHOT.jar
```

后端配置文件：

- `backend/src/main/resources/application.yml`

当前默认数据库配置：

- URL: `jdbc:mysql://localhost:3306/or_sys...`
- 用户名: `root`
- 密码: `root`

如与你本机不一致，请先修改再启动。

## 5. 第三步：启动前端（Yarn）

在新的终端窗口执行：

```bash
cd frontend
npm install -g yarn
yarn install
yarn serve
```

## 7. 账号与登录说明

### 7.1 当前种子数据现状

- `data.sql` 当前默认插入的是患者账号数据（`t_user.user_type` 为 `PATIENT`）。
- 管理员/医生账号通常需要你在本地手动补充或通过管理接口创建。

### 7.2 推荐调试方式

1. 患者角色：直接注册新账号测试挂号流程。  
2. 管理员角色：将某个用户 `user_type` 改为 `ADMIN`。  
3. 医生角色：将某个用户 `user_type` 改为 `DOCTOR`，并在 `t_doctor.user_id` 绑定该用户。

## 8. 常见问题

### 8.1 前端启动报错 `yarn serve` 找不到命令

请确认已在 `frontend` 目录执行，且已拉取最新代码（当前 `package.json` 已提供 `serve` 脚本）。

### 8.2 后端启动后前端提示 403

常见原因：

- 未登录就访问了需要权限的接口（如 `/api/appointment/**`）
- 角色不匹配（患者/医生/管理员权限边界）

### 8.3 数据导入后登录异常

先确认：

1. `data.sql` 是否导入到了 `or_sys`
2. `application.yml` 的数据库连接是否仍指向 `or_sys`

### 8.4 端口占用

Windows 可检查：

```powershell
netstat -ano | findstr :8080
netstat -ano | findstr :5173
```

如需改端口：

- 后端：`backend/src/main/resources/application.yml` -> `server.port`
- 前端：`frontend/vite.config.ts` -> `server.port`，并同步代理 `target`

## 9. 停止服务

- 后端终端：`Ctrl + C`
- 前端终端：`Ctrl + C`
